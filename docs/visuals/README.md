# Визуальные прототипы

Эта папка содержит быстрый UI/UX-контур Lumi. Он нужен для проверки
компоновки, визуального языка и пользовательских сценариев до переноса решений
в продуктовый web-клиент.

## Структура

- [`prototype/`](prototype/) — текущий интерактивный прототип на чистых
  HTML, CSS и JavaScript;
- остальные `*.md` в этой папке — заметки и решения, относящиеся к визуальному
  прототипу.

## Как запустить

Из корня репозитория:

```sh
make prototype-r
```

После этого прототип доступен по адресу <http://127.0.0.1:4173>.

Можно обойтись без `make`:

```sh
python3 -m http.server 4173 --bind 127.0.0.1 --directory docs/visuals/prototype
```

Сборка, backend, Rust, Dioxus, React и установка npm-зависимостей не нужны.

После изменения поведения запустите отдельный browser gate:

```sh
make prototype-e2e
```

Он поднимает тот же статический сервер и не запускает Dioxus или backend.

## Правила работы

1. Прототип использует только статические HTML, CSS, JavaScript и локальные
   ассеты. Все данные и ответы сервисов — заглушки.
2. Здесь допустимы быстрые и намеренно простые реализации. Прототип проверяет
   интерфейс и сценарий, а не продуктовую архитектуру.
3. Значимое визуальное решение сначала описывается в `docs/visuals/*.md`, затем
   при необходимости переносится в каноническую продуктовую или системную
   документацию.
4. Прототип не является источником доменных контрактов и не должен импортировать
   код из `apps/web` или `crates/`.
5. При переносе решения в продукт сохраняются требования к семантике,
   клавиатурной навигации, адаптивности и тестируемости через role locators.

## Текущее состояние

Текущая версия проверяет reader-first направление текущего S1 Web EPUB Reader:
библиотека и чтение разделены, текст является основной поверхностью, а
оглавление, настройки и личные аннотации открываются по запросу. Технические
идентификаторы, экспорт и диагностика остаются доступны контекстно, но не
конкурируют с чтением.

Для проверки поведения без backend добавлены небольшие локальные интеракции:
переход из библиотеки в reader, панели оглавления и заметок, contextual actions
для выделения, настройка типографики и темы, page-like переходы, архивирование
материала и скачивание демонстрационного JSON-экспорта.

Визуальное решение и границы прототипа описаны в
[`reader-first-direction.md`](reader-first-direction.md).
