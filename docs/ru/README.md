# Русская документация

Эта папка содержит русскую рабочую ветку документации: исходные обсуждения,
черновики, заметки и материалы, из которых затем стабилизируется английский
канон разработки.

Каноничная ветка для разработки - [`../en`](../en). Сначала можно обсуждать и
фиксировать позицию на русском здесь, но долговечные продуктовые,
архитектурные, процессные и реализационные решения нужно перевести и закрепить
в соответствующем пути `docs/en`.

Набор Markdown-документов должен совпадать с [`../en`](../en) по относительным
путям. Если документ есть только в одной языковой ветке, во вторую ветку нужно
добавить зеркало, а не удалять исходный документ.

Текущие документы:

- [`vision.md`](vision.md) - продуктовый vision.
- [`early-slices.md`](early-slices.md) - первые реализационные срезы.
- [`system-design/`](system-design/) - принятый target design для `v01`.
- [`adr/`](adr/) - долговечные архитектурные решения.
- [`runbooks/local-dev.md`](runbooks/local-dev.md) - локальный workflow
  разработки.

Временные реализационные планы живут вне языкового зеркала в
[`../tmp-plans`](../tmp-plans). Это tactical working documents для активных
промежуточных срезов разработки. Если в таком плане появляется долговечное
продуктовое, архитектурное или процессное решение, его нужно перенести обратно
в обе языковые ветки, закрепив английскую версию как канон разработки.
